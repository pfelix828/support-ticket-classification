## Classification Design Notes

### Proposed Tier 1 Categories (broad routing)
1. Billing & Payments
2. Account Access & Login
3. Account Management
4. API Technical
5. API Billing & Usage
6. ChatGPT Product
7. ChatGPT Apps (mobile/desktop)
8. Enterprise / Business / Edu
9. GPTs & Custom GPTs
10. Newer Products (Sora, Codex, Atlas)
11. Privacy & Policy
12. Security

### Key classification challenges
- "Account, login and billing" is one help center category but 3+ distinct ticket types
- API rate limit tickets could be billing, technical, or usage — context matters
- Enterprise vs. consumer tickets for same issue route differently
- Newer products have fewer training examples (long-tail)
- Multi-intent tickets common ("charged twice AND API key stopped working")
- Sentiment/urgency affects routing priority
